# Product

## Register

product

## Users

Engineers, demonstrators, developers, and technical evaluators who use the IJT Web Client to connect to OPC UA Industrial Joining Technologies servers, inspect server data, visualize joining results and traces, and validate envelope behavior in a browser. They are usually working in a technical, data-heavy context where correctness, traceability, and fast diagnosis matter more than persuasion.

## Product Purpose

The IJT Web Client is a GUI reference client for visualizing OPC UA IJT server data, events, traces, assets, and results. It helps users understand how IJT data is structured, inspect live or simulated joining process output, and configure or validate trace and envelope workflows without hiding the underlying technical model.

## Brand Personality

Precise, technical, trustworthy.

## Anti-references

Avoid sparse generic admin panels. The interface should not feel empty, anonymous, or disconnected from the industrial joining workflow. It should also avoid decorative AI-style UI patterns that make dense technical data feel less credible.

## Design Principles

- Keep data provenance visible: users should be able to understand where displayed trace, result, and envelope values came from.
- Favor explicit technical controls over magical automation: automated actions must expose enough context to be trusted.
- Preserve workflow continuity: adding controls should not interrupt trace inspection, result comparison, or validation loops.
- Make dense information scannable: hierarchy, grouping, labels, and states should reduce search cost without oversimplifying the model.
- Design for safe iteration: destructive or broad actions need clear labels, predictable scope, and recoverable mental models.

## Accessibility & Inclusion

Target WCAG 2.2 AA where practical. Future UI work should preserve keyboard access, visible focus states, sufficient contrast, usable disabled and empty states, and clear text alternatives for icon-only controls.
